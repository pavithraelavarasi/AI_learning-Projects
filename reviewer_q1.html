Reviewer View
CSS Box Model & Profile Card
Question 1 HTML & CSS Box Model Reverse Engineering
Original Question
1

Objective

To develop CSS Box Model literacy by reverse-engineering an AI-generated personal profile card and making intentional modifications to observe how each property shapes layout and spacing.

Input / Scenario

You have just met someone at a networking event, and you want a quick way for them to remember who you are later. You need a simple visual card that shows your face, name, what you do, and a short note about yourself. Build the simplest card that lets you see this at a glance.

Expectations

E1

Read the AI-generated HTML and CSS code line by line and understand what each part does.

E2

Identify where the Box Model properties — margin, border, padding, and content — appear in the code.

E3

Explain in plain words how each property affects the card's appearance and spacing.

E4

Modify at least one property from each Box Model layer and observe what changes in the browser.

E5

Articulate the difference between padding and margin — without looking it up.

E6

Critique the AI-generated code — identify one design or spacing decision you disagree with, explain why, and improve it.

Reviewer Questions
Post-Completion Review
Comprehension
1
Code Reading Walk me through the HTML structure of your profile card. What elements did you use, and why?
2
Code Reading Pick any three CSS rules from the generated code and explain in plain language exactly what each one does to the card's appearance.
3
Code Reading Is there any part of the code you found confusing or unexpected? What did you do to make sense of it?
Box Model Identification
4
Margin Show me exactly where margin appears in the code. What element does it apply to, and what visual effect does it create?
5
Border Where is border defined in the code? What happens visually if you remove it entirely?
6
Padding Point to where padding is set. Which element does it belong to, and what would happen to the card if you doubled its value?
7
Content Where is the "content" layer visible in your card — in the HTML and in the browser? How does a fixed width or height interact with it?
8
Box Sizing Does the code use box-sizing: border-box? If yes, what does that change about how width is calculated? If not, how does the default model affect your layout?
Explanation
9
Conceptual Without looking anything up — explain the difference between padding and margin in your own words. Can you give a real-life analogy?
10
Conceptual If the card's text looks cramped, which Box Model property would you adjust first — and why?
11
Visual Reasoning If you increase the card's padding to 60px, does the card get wider, the content area shrink, or both? Explain why, based on the Box Model.
12
Spacing Logic What is margin: 0 auto doing in your code, if present? How does it center the card, and what condition must be true for it to work?
Modification & Observation
13
Hands-On What specific change did you make to the margin? What did you observe in the browser, and did it match what you expected?
14
Hands-On What change did you make to the border? Did changing the border width affect the card's total size? Why or why not?
15
Hands-On What change did you make to padding? Show me the before and after in the browser. What shifted, and what stayed the same?
16
Hands-On What change did you make to the content layer — such as width, height, or font size? How did other Box Model layers respond to it?
17
Exploration Did any of your modifications produce an unexpected result? What surprised you, and how did you investigate or fix it?
Articulation
18
Concept Check Imagine explaining the CSS Box Model to a non-technical friend in two sentences. What would you say?
19
Concept Check If a colleague says "just add some padding to push the cards apart," what would you tell them is actually the right approach, and why?
20
Deeper Understanding What is "margin collapse," and did you encounter anything like it while building the card?
Critical Thinking
21
Critique Which specific design or spacing decision in the AI-generated code did you disagree with? Walk me through your reasoning.
22
Improvement Show me the improvement you made. How does the updated version better serve someone who just received your card at a networking event?
23
Reflection Looking back, is there anything in the AI-generated code you would have done differently from the start? What would it be, and what Box Model property is involved?
24
Open-Ended The AI was told to build "the simplest card." Do you think it succeeded? Is there anything it included that you consider unnecessary, or anything important it left out?
Reviewer Note: These questions are designed to surface genuine understanding — not recall. Look for students who can connect what they see in the code to what they see in the browser, and who can explain their own modifications confidently. Struggle with Q9 (padding vs. margin without reference) and Q18 (plain-language explanation) are especially revealing. Probe follow-up on any answer that sounds memorized.
