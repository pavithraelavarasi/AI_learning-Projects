Variables & State

A variable is a named container for a value. State is a programmer's word for data that changes over time — and variables are how JavaScript tracks it.

let vs const — when to use each
let
Value will change. Use for anything that gets updated: scores, toggle states, form fields, counters, positions.
let score = 0;
score = score + 10; // ok
const
Value will never change. Use for fixed config: max limits, API keys, element references, colour values.
const MAX = 100;
MAX = 200; // TypeError!
Scope — where can a variable be seen?
Global scope
let total = 0;  visible everywhere
Function scope (inside a function)
let tax = total * 0.1;  visible here
total is visible here — it was declared outside
tax is NOT visible here — it was declared inside the function
! Avoid var — it has confusing scope rules. In all modern JavaScript, use let for changeable values and const for everything else. Declare variables at the top of the block they belong to.

getElementById vs querySelector
getElementById
Finds one element by its exact id. Fastest method — IDs are unique by definition.

document.getElementById("output")

Returns the element, or null if not found.
querySelector
Accepts any CSS selector. More flexible but slightly slower.

document.querySelector("#output")
document.querySelector(".primary")
document.querySelector("button")
JavaScript
// Store references at the top of your script — once const outputEl = document.getElementById("output"); const actionBtn = document.getElementById("action-btn"); // querySelector: any CSS selector works const footer = document.querySelector(".site-footer"); const allBtns = document.querySelectorAll("button"); // NodeList
! Timing matters: if your script runs before the HTML has rendered, getElementById returns null. Always place <script> tags at the bottom of <body>, or listen for DOMContentLoaded.

Conditional Logic

Conditionals let your code make decisions. Depending on the current state — a number, a boolean, a string — different paths execute.

JavaScript
// Ternary — compact single-line alternative const label = score >= 90 ? "Pass with distinction" : "Standard pass"; // Logical operators in conditions if (score >= 50 && attempts < 3) { … } // AND — both must be true if (isAdmin || isModerator) { … } // OR — either is enough if (!isLoggedIn) { … } // NOT — inverts the value // Strict equality — always use === not == if (status === "active") { … } // type + value must match
◆ Always use === (strict equality) instead of ==. The loose version does type coercion: "5" == 5 is true, which is almost never what you want. Strict equality requires both value and type to match.
